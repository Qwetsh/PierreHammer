1. Vue d'ensemble du pipeline

Le simulateur calcule l'espérance mathématique (moyenne statistique) des dégâts infligés par une unité attaquante à une unité défensive. Il ne fait pas de simulation Monte-Carlo (pas de lancers aléatoires) : chaque étape multiplie une probabilité pour obtenir le résultat moyen attendu.

Attaques → Touches → Blessures → Sauvegardes → Dégâts → FnP → Pertes

Chaque étape réduit le flux en le multipliant par la probabilité de réussite (ou d'échec pour les sauvegardes). Les mots-clés d'arme et les capacités créent des branches parallèles (ex : Lethal Hits contourne le jet de blessure, Devastating Wounds contourne la sauvegarde).

2. Fonctions utilitaires

2.1 Notation de dés → Valeur moyenne

Toutes les valeurs de jeu (Attaques, Dégâts, etc.) sont converties en leur espérance mathématique :

// Formule générale : NdS+B
moyenne = N × (S + 1) / 2 + B

// Exemples :
"3" → 3     "D6" → 3.5     "D3" → 2
"2D6" → 7     "D6+1" → 4.5     "2D3+3" → 7

2.2 Probabilité de réussite sur D6

Pour un seuil T (ex : 3+ signifie réussir sur 3, 4, 5 ou 6) :

P(réussite) = (7 − T) / 6

// Le seuil est toujours borné à [2, 6]
// Un 1 naturel est toujours un échec (seuil min = 2)
// Un 6 naturel est toujours une réussite (seuil max = 6)

2.2 Cas spécial : Torrent

Les armes Torrent touchent automatiquement : toutes les attaques sont des touches. Aucun dé n'est lancé, donc aucun coup critique n'est généré (pas de Sustained Hits, pas de Lethal Hits).

2.3 Calcul des touches

// Probabilité de coup critique (6 naturel)
P_crit = 1/6

touches = attaques × P(seuil)
coups_critiques = attaques × P_crit

// Sustained Hits N : chaque critique génère N touches supplémentaires
si Sustained Hits : touches += coups_critiques × N

// Lethal Hits : les critiques deviennent des blessures automatiques
si Lethal Hits :
  blessures_létales = coups_critiques
  touches −= coups_critiques // retirées du flux normal
Lethal Hits retire les coups critiques du flux de touche et les injecte directement en blessures, contournant complètement l'étape 3 (jet de blessure).

3 Jet de blessure (Wound Roll)

3.1 Seuil de blessure

seuil_blessure = table_F_vs_E(Force_arme, Endurance_cible)

// Modificateurs :
si Lance && charge : seuil −= 1
seuil −= modificateurs_attaquant(wound)
seuil = clamp(seuil, 2, 6)

3.2 Anti-X (seuil critique modifié)

// Normalement, un coup critique blessure = 6 naturel
seuil_crit_blessure = 6

// Anti-X abaisse ce seuil (ex: Anti-Infantry 4+)
si Anti-X : seuil_crit_blessure = min(6, meilleur_seuil_anti)

P_crit_blessure = (7 − seuil_crit_blessure) / 6

3.3 Twin-Linked (relance des échecs)

P_blessure_base = (7 − seuil) / 6

// Sans Twin-Linked :
P_effective = P_blessure_base

// Avec Twin-Linked (relance tous les échecs) :
P_échec = 1 − P_blessure_base
P_effective = P_base + P_échec × P_base
P_effective = P_base × (2 − P_base)

3.4 Calcul des blessures

blessures = touches × P_effective + blessures_létales

// Devastating Wounds : les critiques deviennent des blessures mortelles
si Devastating Wounds :
  blessures_crit = touches × P_crit_blessure
  blessures_mortelles += blessures_crit × moy(Dégâts_arme)
  blessures −= blessures_crit // retirées du flux normal
Devastating Wounds convertit les blessures critiques en dégâts mortels qui contournent entièrement l'étape 4 (sauvegarde). Les dégâts mortels sont égaux au nombre de blessures critiques × la valeur moyenne de dégâts de l'arme.

4 Jet de sauvegarde (Save Roll)

4.1 Sauvegarde d'armure modifiée

svg_modifiée = Svg + |PA|

// Couvert (tir uniquement, si l'arme n'ignore pas le couvert) :
si couvert && tir && !ignore_couvert :
  svg_modifiée −= 1 // améliore la svg (plus bas = mieux)

4.2 Sauvegarde invulnérable

// La meilleure entre le profil et les capacités
meilleure_invuln = min(invuln_profil, invuln_capacité)

// On utilise la meilleure sauvegarde disponible
si meilleure_invuln > 0 ET meilleure_invuln < svg_modifiée :
  seuil_svg = meilleure_invuln // l'invuln est meilleure
sinon :
  seuil_svg = svg_modifiée

seuil_svg = clamp(seuil_svg, 2, 6)
La sauvegarde invulnérable n'est jamais modifiée par la PA. C'est pour cela que les armes à forte PA rendent l'invuln plus intéressante.

7 Estimation des pertes

PV_figurine = Endurance du profil défenseur (W)

pertes_estimées = min(nb_défenseurs, dégâts_finaux / PV_figurine)
Le résultat est plafonné au nombre de figurines dans l'unité cible. Le simulateur suppose que les dégâts excédentaires sur une figurine tuée ne sont pas perdus (moyenne statistique), ce qui est une simplification par rapport aux règles réelles où l'overflow sur une figurine est perdu.

8. Mots-clés d'arme et leurs effets

Mot-cléEtape affectéeEffet dans le calcul
Torrent Touche Toutes les attaques touchent automatiquement. Pas de critiques.
Heavy Touche Si stationnaire : seuil de touche −1 (meilleur).
Blast Attaques +1 attaque par tranche de 5 figurines dans la cible.
Rapid Fire N Attaques +N attaques à demi-portée.
Sustained Hits N Touche Chaque 6 naturel génère N touches supplémentaires.
Lethal Hits Touche → Blessure Les 6 naturels deviennent des blessures automatiques (sautent le jet de blessure).
Twin-Linked Blessure Relance tous les jets de blessure ratés. Peff = P(2 − P).
Anti-X N+ Blessure Les blessures critiques se déclenchent sur N+ au lieu de 6+.
Devastating Wounds Blessure → Dégâts Les blessures critiques deviennent des dégâts mortels (contournent la svg).
Lance Blessure Si charge : seuil de blessure −1.
Melta N Dégâts À demi-portée : +N dégâts par blessure.
Ignores Cover Sauvegarde Le défenseur ne bénéficie pas du bonus de couvert.

9.2 Conditions

Les modificateurs peuvent être conditionnels :

  • ranged_only — ne s'applique qu'aux armes de tir
  • melee_only — ne s'applique qu'aux armes de mêlée
  • vs_keyword:X — ne s'applique que contre les cibles avec le mot-clé X

10. Exemples concrets

Exemple 1 : Bolters classiques contre Intercessors

// 10 Space Marines tirent avec des Bolt Rifles
// A:2, CT:3+, F:4, PA:-1, D:1 vs T:4, Svg:3+, PV:2

Attaques : 2 × 10 = 20
Touches : 20 × P(3+) = 20 × 4/6 = 13.33
Blessures : F4 vs T4 → 4+ → 13.33 × 3/6 = 6.67
Sauvegarde : Svg 3+ avec PA-1 → 4+ → P(échec) = 3/6
  Non sauvegardées : 6.67 × 3/6 = 3.33
Dégâts : 3.33 × 1 = 3.33
Pertes : 3.33 / 2 = 1.67 figurines

Exemple 2 : Lascannon avec Devastating Wounds

// 1 Lascannon : A:1, CT:3+, F:12, PA:-3, D:D6+1
// Devastating Wounds, vs T:10, Svg:2+, Invuln:4+, PV:14

Attaques : 1
Touches : 1 × 4/6 = 0.667
  Critiques : 1 × 1/6 = 0.167

Blessures : F12 vs T10 → 3+ → P = 4/6
  Blessures normales : 0.667 × 4/6 = 0.444
  Blessures critiques (Devastating) : 0.667 × 1/6 = 0.111
  Blessures mortelles : 0.111 × moy(D6+1) = 0.111 × 4.5 = 0.50
  Blessures restantes : 0.444 − 0.111 = 0.333

Sauvegarde : Svg 2+ avec PA-3 → 5+, mais Invuln 4+ est meilleur
  Non sauvegardées : 0.333 × 3/6 = 0.167

Dégâts : 0.167 × 4.5 + 0.50 = 1.25 dégâts
Pertes : 1.25 / 14 = 0.09 figurine

Exemple 3 : Twin-Linked + Anti-Infantry 4+

// Flamestorm cannon : A:D6, CT:N/A (Torrent), F:6, PA:-2, D:2
// Twin-Linked, Anti-Infantry 4+, vs T:3, Svg:5+, PV:1, 10 figurines

Attaques : moy(D6) = 3.5
Touches : Torrent → 3.5 (auto-touché)
  Pas de critiques (Torrent = pas de dé)

Blessures : F6 vs T3 → 2+ → P = 5/6
  Twin-Linked : Peff = 5/6 × (2 − 5/6) = 5/6 × 7/6 = 35/36 = 97.2%
  Blessures : 3.5 × 35/36 = 3.40

Sauvegarde : Svg 5+ avec PA-2 → 7+ (pas de svg possible)
  Non sauvegardées : 3.40

Dégâts : 3.40 × 2 = 6.81
Pertes : 6.81 / 1 = 6.81 figurines (plafonné à 10)
